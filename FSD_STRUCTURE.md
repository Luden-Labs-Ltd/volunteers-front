# Структура проекта по FSD (Feature-Sliced Design)

## Слои (Layers)

### `app/`

Инициализация приложения, роутинг, провайдеры

- `providers/` - провайдеры React
- `index.tsx` - точка входа

### `pages/`

Полные страницы приложения

- `auth/` - страница авторизации
- `tasks/` - страница списка задач
- Каждая страница имеет структуру: `ui/` сегмент

### `widgets/`

Крупные самодостаточные блоки функциональности

- `task-list/` - виджет списка задач
- Каждый виджет имеет структуру: `ui/` сегмент

### `features/`

Переиспользуемые бизнес-фичи (действия пользователя)

- `create-task/` - создание задачи
- `respond-to-task/` - отклик на задачу
- Каждая фича имеет структуру: `ui/`, `model/` сегменты

### `entities/`

Бизнес-сущности (данные и их отображение)

- `task/` - сущность задачи
  - `model/` - типы, схемы данных
  - `api/` - API методы
  - `ui/` - UI компоненты
- `user/` - сущность пользователя
  - `model/` - типы данных

### `shared/`

Переиспользуемые компоненты без бизнес-логики

- `ui/` - UI компоненты (Button, Input)
- `api/` - базовый API клиент
- `lib/` - утилиты (formatDate, formatPhone)
- `assets/` - переиспользуемые ресурсы (изображения, иконки)

## Сегменты (Segments)

Каждый слой/slice делится на сегменты по техническому назначению:

- `ui/` - компоненты интерфейса
- `api/` - взаимодействие с бэкендом
- `model/` - типы, схемы, бизнес-логика
- `lib/` - утилиты и хелперы
- `config/` - конфигурация
- `assets/` - изображения и другие ресурсы (опционально, внутри slice)

## Правила импортов

1. Модуль может импортировать только с нижних слоев:

   - `pages` → `widgets`, `features`, `entities`, `shared`
   - `widgets` → `features`, `entities`, `shared`
   - `features` → `entities`, `shared`
   - `entities` → `shared`
   - `shared` → ничего

2. Slices не могут использовать другие slices на том же слое

3. `app` и `shared` слои не имеют slices, делятся напрямую на segments

## Пример структуры entity

```
entities/
  task/
    model/
      types.ts          # TypeScript типы
      index.ts          # Public API
    api/
      task-api.ts       # API методы
      index.ts          # Public API
    ui/
      task-card/
        ui.tsx          # UI компонент
        index.ts        # Public API
      index.ts          # Public API
```
